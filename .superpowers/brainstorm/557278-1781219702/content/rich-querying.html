Rich Querying & Filtering

Combining Metadata Filters with Vector Search

To support real-world apps, we need to filter by more than just ID or similarity. We need to answer: "Find similar images created in the last 24 hours in the 'memories' collection."

A

Fluent Query Builder

Clean SDK-side API. The Store receives a structured Query object. Best for Go/JS developers.

db.Collection("images")
  .Where("wall_time", ">", yesterday)
  .Search(vector, 10)
B

SQL-Lite String Parser

Accept a subset of SQL. Familiar for users, but requires a parser in both Go and WASM. More flexible for complex logical chains.

SELECT * FROM images 
WHERE wall_time > ? 
ORDER BY similarity(?) LIMIT 10
Store Interface Addition
type Query struct {
    Collection string
    Filters    []Filter
    Vector     []float32
    Limit      int
}

type Store interface {
    // ...
    Query(q Query) ([]Record, error)
}
