Implementing Namespaces (Collections)

You chose Explicit Fields. Here is how that changes the core data model.

Current (Flat)
type Record struct {
    ID        string
    Type      DataType
    Payload   []byte
    // ...
}

type Store interface {
    Get(id string) (Record, error)
    Delete(id string) (Record, error)
    // ...
}
Proposed (Collections)
type Record struct {
    Collection string // New field
    ID         string
    Type       DataType
    Payload    []byte
    // ...
}

type Store interface {
    // API changes to require collection
    Get(col, id string) (Record, error)
    Delete(col, id string) (Record, error)
    // ...
}

How should we handle the transition?

Adding a collection parameter to every method is a breaking change for the Store interface.

A

Strict Collections

Every read/write MUST specify a collection. Data is physically isolated in the backend (e.g. separate bbolt buckets).

B

Default Collection

If collection is empty, use a "default" namespace. Allows easier migration but adds "special case" logic.
