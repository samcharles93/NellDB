Binary Wire Protocol

Reducing bandwidth for vectors and image payloads

JSON is currently our only wire format. It's readable, but it's 2-3x larger than binary and requires expensive Base64 encoding for []byte and []float32 fields.

A

CBOR (Concise Binary Object Representation)

Binary JSON. Easiest to implement. No schema files required. Native support for byte arrays. Great balance of speed and simplicity.

B

Protobuf

The Industry Standard. Smallest payloads and fastest parsing. Requires .proto files and code generation. Very strict, which is good for stability.

C

Custom Flat-Buffer

Zero-copy performance. Best for WASM/Go interaction. We define a fixed-width header (8-byte HLC, etc.) and append the payload. Hardest to maintain but fastest possible sync.
