Optimized Vector Indexing

Sub-millisecond similarity search at scale

Currently, Nell performs a Linear Scan for similarity searches (O(N)). This works for 1,000 documents, but hangs at 100,000. We need an Approximate Nearest Neighbor (ANN) index.

A

HNSW-lite (Hierarchical Navigable Small Worlds)

The Gold Standard. Fastest query speed and best accuracy. We can implement a "lite" version that is memory-mapped and works well in both Go and WASM.

B

IVF-Flat (Inverted File Index)

Faster Training. Simpler to implement than HNSW. Clusters vectors into "cells." During search, only a few cells are scanned. Good for lower memory environments.

C

Product Quantization (PQ)

Memory Optimized. Compresses 128 floats into a few bytes. Can be combined with A or B to store millions of vectors in a tiny memory footprint.
