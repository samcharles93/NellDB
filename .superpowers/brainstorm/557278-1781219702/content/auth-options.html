Auth & Multi-tenancy

How should we scope access to your data?

Now that we have Collections, we can think about how to secure them. Nell is a peer-to-peer system, so auth happens at the sync layer.

A

Global Bearer Token

Simple static token for the whole server. Good for private meshes.

B

Per-Collection Keys

Each collection has its own read/write key. Allows scoping sync to specific datasets.

C

OIDC / JWT

Integrate with external auth providers. Better for multi-user apps.

Sync Request Example
POST /sync/pull?col=messages
Authorization: Bearer your-token
